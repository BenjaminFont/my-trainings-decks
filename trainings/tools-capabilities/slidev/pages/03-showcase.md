<!--
Working Slot „MCP" — Vorgehen. Drei Phasen:

Phase 1 — Entdecken:
• Aufgabe: Schauen, welche MCP-Server es noch für die eigene Arbeit gibt (siehe Ressourcen-Folie davor).
• Welche davon machen für die eigene Rolle Sinn (Dev / QA / PO-PM-RE)?
• Anlaufstellen: die eingesetzten Tools (Jira/Rovo, GitLab …) sowie MCP-Verzeichnisse und die jeweilige Doku.

Phase 2 — Plenum / Teilen:
• Kurz teilen: Welche MCP-Server wollt ihr euch genauer anschauen und ausprobieren — und warum gerade die?

Phase 3 — Einrichten & Testen:
• Den gewählten MCP-Server in Claude Code einhängen.
• An einer kleinen, realistischen Aufgabe austesten, ob er den erwarteten Nutzen bringt.

Moderation: Phase 1 als stille/kleingruppige Recherche, Phase 2 als kurze Plenumsrunde, Phase 3 als Hands-on mit Zeitbox; danach ggf. kurze Review-Runde.
